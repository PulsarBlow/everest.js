Simple Test EverestJs samples

This sample is demonstrating how to use Everest REST API client in the browser, using the non-AMD module way (old-school fashion global variable).

EverestJs is also usable as an AMD-module (with a module loader like require.js).

Check the RESTAPI Explorer sample to see how it works.

EverestJs global variable name is : window.ê or window.everest
You can use both. But ê is much cooler, isn't it ?

RestApi Client

The RestApi client is a high level RESTful abstraction: ê.RestApiService

Create a new client instance 
var restApi = new ê.RestApiService({

    // required - Base url of your REST Api
    baseUrl: "https://api.github.com",

    // optional - Data format (aka content-type)
    dataFormat: "json",     // ("json" | "xml" | "atom" | "form") - default : "json",

    // optional - Data encoding (aka charset)
    dataEncoding: "utf-8", // default: "utf-8",

    // optional - Should GET and HEAD requests be cached by the browser ?
    useCache: false,        // (true | false) - default: false

    // optional - Force the client to communicate in HTTPS only, whatever protocol is
    // provided in baseUrl
    strictSSL: false        // (true | false)  - default: false
});
Read a resource 
//
restApi.read("/repos/PulsarBlow/everestjs")
.done(function(data) {
    console.log("restApi response", data);
})
.fail(function() {
    console.log("restApi response failed", arguments);
})
.always(function() {
    console.log("resApi call completed");
});

Http Client

The RestApi client uses this Http abstraction to execute low level HTTP requests.

The HttpClient is also part of the public API, so you can use it for any purpose: ê.HttpService

You can either use it as an instance or with its static methods.

// Everest provides predefined constants you can use.
var contentTypes = ê.constants.http.contentTypes;
var charsets = ê.constants.http.charsets;

// Use the constructor function to provide your configuration
var httpClient = new ê.HttpClient({

    // Define the "Accept" HTTP header for all your requests
    accept: contentTypes.JSON,

    // Defines the "Accept-Charset" HTTP header for all your requests
    acceptCharset: charsets.UTF8,

    // Defines the "Content-Type" HTTP header for all your requests
    contentType: contentTypes.JSON,

    // Defines the charset which will be appended to your "Content-Type" HTTP header
    // eg: application/json; charset=utf-8
    contentTypeCharset: charsets.UTF8,

    // Should GET and HEAD requests cached by the browser ?
    cache: false,

    // All request will be made over HTTPS only
    strictSSL: false    // (true | false) - default: false
});
